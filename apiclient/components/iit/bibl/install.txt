Установка библиотеки на Centos
рабочий вариант - eusphpei.64.7.4.4.tar

Для встановлення бібліотеки для web-серверу Apache, де PHP завантажується як модуль Apache необхідно:
    • створити каталог
        sudo mkdir /usr/lib64/php/eusphpe_extension
    • разархівувати бібліотеки з архіву eusphpei.tar до каталогу:
        sudo cp eusphpei.64.7.4.4.tar /usr/lib64/php/eusphpe_extension
        cd /usr/lib64/php/eusphpe_extension
        tar -x–f eusphpei.64.7.4.4.tar (или подругому, неважно)
    • до файлу /etc/profile додати:
        sudo vi /etc/profile
        -> export LD_LIBRARY_PATH="/usr/lib64/php/eusphpe_extension"
    • до файлу /etc/php.ini додати:
        sudo vi /etc/php.ini
        extension=/usr/lib64/php/eusphpe_extension/eusphpe.so
    • перзапустити apache:
        sudo systemctl restart httpd.service

        3.3 Параметри роботи бібліотеки
        Параметри, що використовуються бібліотекою,
    в файлі конфігурації з використанням кодування CP-1251.
        В ОС Linux/Unix, SunOS параметри, що використовуються бібліотекою,
    зберігаються у файлі конфігурації osplm.ini
        Пошук файла конфігурації в цих системах виконується в наступній послідовності:
            • в каталогах, що внесені у змінну оточення LD_LIBRARY_PATH (ОС Linux, Sun OS, FreeBSD), LIBPATH (ОС AIX);
            • в поточному каталозі;
            • в каталозі з програмою.
        У випадку, якщо не знайдено файл конфігурації osplm.ini,
    бібліотекою буде виконано пошук файлу конфігурації ospcu.ini у послідовності, що вказана вище.
        Якщо файли конфігурації не знайдено, файл osplm.ini буде створено в каталозі з програмою
        Якщо права запису до файлу конфігурації відсутні, параметри буде збережено у внутрішньому стані бібліотеки.
        Якщо зміни вносяться безпосередньо до файл конфігурації або реєстру (без використання відповідних функцій бібліотеки),
для застосування встановлених параметрів необхідно переініціалізувати бібліотеку (перезапустити web-сервер, інтерпретатор PHP).
